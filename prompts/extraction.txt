source_type values:
- "fram_drawing"  — FRAM / Kjættingfabriken single-component technical drawing
- "assembly"      — assembly drawing with a bill-of-materials (stykkliste) table listing component parts
- "third_party"   — product page from another brand (Tigrip, Crosby, etc.)
- "catalog_row"   — single row extracted from a tabular catalog page

Available categories (use exactly one, unchanged):
Kjetting, Koplingselementer, Sjakler, Løkker, Kroker, Løftepunkt,

Extraction rules:
- Return ONLY the JSON array — no prose, no markdown code fences
- Always return an array, even for a single product
- wll_tonnes: convert from kg if needed (e.g. 2000 kg capacity → 2.0)
- components: for assembly drawings, extract EVERY row from the BOM/parts table including part_number, description, qty, and length if listed
- Catalog pages with multiple product rows: return one object per row (each gets its own item_number / model etc.)
- Third-party catalog pages: use vendor_part_number for the Art.-No., item_number is null
- tags: include product type, material, capacity range, application — lowercase, Norwegian and/or English
- If a field is not visible or not applicable, set it to null (never omit a field)
- For multi-page drawings of the same product (e.g. page 1 of 2 / page 2 of 2), return ONE product object using the first page number; merge all information from all pages
